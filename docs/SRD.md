# 📄 System Requirements Document (SRD)

---

# 1. System Architecture

The system follows a **three-tier architecture**:

### Frontend Layer

Handles the user interface and user interactions.

### Backend Layer

Processes requests, manages business logic, and handles APIs.

### Database Layer

Stores information related to users, businesses, and reviews.

---

# 2. Software Requirements

### Frontend

* HTML
* CSS
* JavaScript
* React (optional)

### Backend

* Python (Flask / Django / FastAPI)

### Database

* MySQL
* PostgreSQL
* Supabase (optional)

### APIs

* Google Maps API or Mapbox API for location display

---

# 3. Hardware Requirements

### Client Side

* Smartphone
* Laptop or Desktop
* Internet connection

### Server Side

* Cloud server or hosting platform (AWS, Firebase, etc.)

---

# 4. System Modules

### 1. User Module

* User registration and login
* Business discovery
* Reviews and ratings

---

### 2. Business Owner Module

* Business registration
* Storefront creation
* Business profile management

---

### 3. Discovery Module

* Map-based business discovery
* Category filtering
* Search functionality

---

### 4. Review Module

* User reviews
* Ratings system
* Community feedback

---

### 5. Admin Module

* Manage businesses
* Monitor reviews
* Remove fake listings or spam

---

# 5. Database Entities

The main database tables include:

* Users
* Businesses
* Reviews
* Ratings
* Categories

---

# 6. Security Requirements

* Secure login authentication
* Data validation
* Protection against fake reviews
* Role-based access control

---

# 7. Future Enhancements

* AI-powered recommendations
* Multi-language support
* Business analytics dashboard
* Integration with digital payment systems
